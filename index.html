What are they anyway?

Understanding the difference between the tool and the platform.

Git

Git is a Version Control System (VCS). It is a software you install on your local computer to track changes in your files over time. It allows you to revert back to previous states, branch off to try new features, and merge changes together.

  • Local tool
  • Command-line based
  • Tracks file history

GitHub

GitHub is a cloud-based hosting service that lets you manage Git repositories. It acts as a central hub where developers can upload their local Git repositories, share them with others, and collaborate on open-source or private projects.

  • Cloud service
  • Web interface
  • Collaboration hub

Getting Started

Install Git on your machine to begin.

Download the official Git for Windows installer.

Download Git Bash

Run the `.exe` file and follow the standard installation prompts (the default settings are recommended for beginners).

The easiest way is to use Homebrew in your terminal:

brew install git

For Debian/Ubuntu-based distributions:

sudo apt-get update
sudo apt-get install git

Initial Configuration

After installing, set up your identity (this will be attached to your commits):

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Common Mistakes & Fixes

"Updates were rejected because the remote contains work that you do not have locally"

Cause: Someone else pushed changes to the GitHub repository, and you are trying to push your own changes without fetching theirs first.

Fix: Pull the latest changes before pushing.

git pull origin main
git push origin main

"fatal: not a git repository"

Cause: You're trying to run a git command in a folder where Git hasn't been initialized yet.

Fix: Initialize git in that folder first.

git init

Committed to the wrong branch?

Fix: You can quickly undo the last commit but keep your file changes by doing:

git reset HEAD~1

Then switch to the right branch and commit again.
